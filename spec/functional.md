# Функціональні вимоги: Гра "Морський бій"

**Версія:** 1.0 | **Платформа:** Android 13+ | **Тип:** 2D мультиплеєр (2 гравці)

## 1. Загальний опис

Мобільний додаток "Морський бій" для двох гравців, кожен на своєму Android 13+. Гравці з'єднуються через мережу за посиланням/кодом, переданим через зовнішній месенджер (Viber, Telegram). Класичний морський бій без міни.

**Глосарій:** *Хост* — створює сесію; *Гість* — приєднується; *Карта* — поле з кораблями; *Хід* — постріл по клітинці супротивника.

## 2. Підключення гравців (FR-CN)

| ID | Вимога |
|---|---|
| 01 | При першому запуску гравець вводить **нік** (3–16 символів, unicode; не лише пробіли). Без реєстрації. Зберігається локально. |
| 02 | Хост створює сесію → отримує унікальне посилання/код. |
| 03 | Хост передає код гостю через зовнішній месенджер. Гість вводить код → підключається. |
| 04 | Статус підключення видимий обом ("очікування" / "підключено"). |
| 05 | **Тайм-аут підключення гостя — 60 сек.** Інакше "Сесія недоступна". |
| 06 | **Тайм-аут перепідключення — 60 сек.** Перевищення → достроковий вихід (FR-GP-13). |
| 07 | **Згортання додатка = вихід з гри.** Стан партії не зберігається. |

## 3. Вибір карти (FR-MP)

Кожен обирає тип; при збігу — використовується, при різниці — система **випадково** обирає один із двох варіантів. Без міни.

| Карта | Поле | Кораблі (палубність×к-сть) | Кораблів | Цілі | Тривалість |
|---|---|---|---|---|---|
| Мала | 8×8 | 3×1, 2×2, 1×3 | 6 | 10 | 3–5 хв |
| Середня | 10×10 | 4×1, 3×2, 2×3, 1×4 | 10 | 20 | 8–12 хв |
| Велика | 12×12 | 5×1, 4×2, 3×3, 2×4, 1×5 | 15 | 30 | 15–25 хв |

**Баланс:** мала без 4-палубника (швидкість); середня — канон; велика додає 5-палубник (стратегія). Обмеження екрана: 12×12 — максимум для портретної орієнтації (клітинка ~28 dp).

## 4. Розстановка кораблів (FR-SP)

| ID | Вимога |
|---|---|
| 01 | Розстановка через **drag-and-drop** іконок. |
| 02 | Валідація: кораблі не перетинаються та не торкаються (включно з діагоналлю). |
| 03 | Кнопки: **"Авторозстановка"** (валідна випадкова за 1 клік), **"Очистити"**, **"Повернути"** (зміна орієнтації виділеного), **"Готовий"**. |
| 04 | Невалідне обертання блокується з візуальним сигналом. |
| 05 | **Після "Готовий" редагування заблоковано** (підтвердження перед натисканням). Очікування супротивника. |

## 5. Ігровий процес (FR-GP)

| ID | Вимога |
|---|---|
| 01 | Гра стартує, коли обидва позначили готовність. Перший хід — **випадково**. |
| 02 | Хід — постріл у клітинку супротивника. **При влучанні гравець продовжує** до першого промаху. Після промаху — хід супротивника. |
| 03 | Корабель **потоплений**, коли всі його клітинки підбиті → автоматично позначаються "промахи" по дотичних клітинках. |
| 04 | **Форма потопленого корабля не розкривається**: показуються лише окремі влучання + дотичні промахи. |
| 05 | **Таймер ходу — 5 хв.** Після спливання — автоматичний пропуск ходу. Видимий обом. |
| 06 | **Багаторівневі попередження таймера:** за 60 сек — звук + жовтий колір; за 30 сек — звук + червоний; за 10 сек — миготіння + вібрація. |
| 07 | **Кнопка "Здатися"** (з підтвердженням) → завжди поразка для того, хто здається (нічия неможлива). |
| 08 | **Панель історії ходів** (бічна панель/розкривайка): [нік] — [координата] — [влучання/промах/потоплено], від нового до старого. Зберігається лише в партії. |
| 09 | Гра завершується, коли в одного гравця всі кораблі потоплені → інший переможець. |
| 10 | **Достроковий вихід** (вихід, обрив >60 сек, згортання): переможець за **кількістю влучань**; рівність → **нічия**. Зараховується в статистику. |

## 6. Завершення гри та реванш (FR-EG)

| ID | Вимога |
|---|---|
| 01 | Екран результату: перемога / поразка / нічия. |
| 02 | **Розширена статистика сесії** (за ніком): перемоги/поразки/нічиї; влучання/промахи; **точність (%)**; **середній час ходу**; потоплені кораблі. Лише поточний сеанс. |
| 03 | **"Зіграти ще раз"** → **переможець обирає карту**, супротивник підтверджує. При нічиї — стандартний вибір (FR-MP). |
| 04 | **"Завершити сесію"** → розрив з'єднання, головний екран. |

## 7. UX та дизайн (FR-UI)

| ID | Вимога |
|---|---|
| 01 | Графіка **2D**, портретна орієнтація. |
| 02 | **Розкладка ігрового екрана:** велике поле супротивника (для стрільби) + **міні-карта свого поля** вгорі. |
| 03 | **Постріл — double tap:** 1-й tap наводить приціл (можна переміщати), 2-й tap по тій же клітинці — постріл. |
| 04 | Звуки: постріл, попадання, потоплення, перемога, поразка, фонова музика. |
| 05 | Анімації: постріл, влучання, **вибух при потопленні**. |
| 06 | **Вібрація** при влучанні/потопленні (вимикається в налаштуваннях). |
| 07 | Видимий індикатор поточного гравця та таймер ходу. |
| 08 | **Без внутрішньоігрового чату** — комунікація через зовнішні месенджери. |

## 8. Налаштування (FR-ST)

Доступні з головного меню; зберігаються локально, застосовуються миттєво:
гучність звуку (0–100%), гучність музики (0–100%, окремо), вібрація (вкл/викл), мова, зміна ніку.

## 9. Нефункціональні вимоги (FR-NF)

- **Платформа:** Android 13+.
- **Мови першого релізу:** українська (за замовчуванням), англійська. Архітектура i18n підтримує додавання інших мов у наступних релізах.
- **Затримка ходу** ≤ 1 сек у нормальній мережі.
- Деталі мережевого протоколу — у технічній документації.
- **Статистика** — лише в межах сесії; **нік + налаштування** — персистентно локально.
