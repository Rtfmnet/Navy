# Navy — Phase 1 Test Suite

## Prerequisites

- **Windows 11** (or Windows 10 1809+)
- **.NET 8 SDK** installed — verify with: `dotnet --version` (must start with `8.`)
- No Unity Editor, Firebase, Java, Node.js, Docker, or Android SDK required.

## Quick Start

```powershell
powershell -ExecutionPolicy Bypass -File tests\RUN_TESTS_SET1.ps1
```

Or from inside the `tests/` directory:

```powershell
powershell -ExecutionPolicy Bypass -File RUN_TESTS_SET1.ps1
```

## What the Script Does

1. Finds `dotnet.exe` and verifies it is .NET 8.x
2. `dotnet restore` — restores NuGet packages
3. `dotnet build -c Release` — builds all projects (zero warnings allowed)
4. `dotnet test` — runs all tests with Coverlet code coverage
5. Parses `coverage/*.cobertura.xml` and prints a summary table
6. Prints test counts: Passed / Failed / Skipped / Duration
7. Exits with code 1 if any test failed or any target assembly is below its coverage threshold
8. Waits for `ENTER` before closing

## Reading the Results

| Colour | Meaning |
|--------|---------|
| Green  | Coverage ≥ 90 % |
| Yellow | Coverage 80–89 % |
| Red    | Coverage < 80 % or test failure |

Coverage targets:
- `Navy.Core.*` — ≥ 95 % line coverage
- `Navy.Data.Firebase.Dto.*` — ≥ 95 % line coverage
- Aggregate (all target assemblies) — ≥ 90 %

## Source Files Under Test

The test projects link directly to `unity/Assets/Scripts/Core/**/*.cs` and
`unity/Assets/Scripts/Data/Firebase/Dto/**/*.cs` via `<Compile Include>` globs.
**No files are copied.** Edits to Unity source are reflected immediately in the next build.

## Out of Scope (Phase 1)

- Firebase integration (real or emulated)
- Unity Editor, Presentation/, Infrastructure/ scripts
- Android build / PC2 steps

See `AUTOFIX_LOG.md` for Phase 2 deferred items.
